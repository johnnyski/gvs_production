Introduction

This package developed by the TRMM Office is in GVBOX.  It contains the product generation programs that automate the execution of level 1, 2 and/or 3 (excluding 2A-56, and 2A-57) of the Ground Validation Software (GVS) production system. The programs can process any data set recognized by level_1 (however, 2A-53-w can process data from Melbourne and Darwin only). Raw Darwin data set is assumed to contain tarred file(s) of compresssed file(s). Other raw data sets are assumed to contain compress/non-compessed file(s). The programs to schedule production for level 1, level 2, and level 3 are all_tape_level_1,all_tape_level_1n2, all_tape_2A-52, and level_3. The programs are named in a self explainatory manner.  There is a graphical user interface program for scheduling the execution of level 1, 2, and/or 3 programs; click on gvs_gui to download that software package.

Description

all_tape_level_1n2:

all_tape_level_1n2 generates level I and/or level II products for an entire 8mm input tape. Additionally, this program can process data from disk or CD. For non-Darwin data processing, the disk and CD organizations do not differ. For Darwin, the CD organization differs from when it is staged from 8mm tapes to disk. The difference is minor because there is not enough space on a CD to hold an entire Darwin tape, so, several CD's contain the data from one Darwin 8mm tape.

all_tape_level_1n2 calls all_tape_level_1 to generate level_1 products. Output files from level I are kept on disk and used as input to the level II programs. Any combination of products for level I and II may be specified on the command line. A directory, gvs_<product>_<tapeid>, will be created for each product. The options file, specific to each product, will be copied to this directory to keep the products seperate from the execution of the other programs that may be running.

Note:

  1. Level I products must reside on disk when running only level II programs.
  2. Can run level 1 from tape, CD, or disk. When data is from CD or disk, type the command all_tape_level_1, and examine the NOTES section w.r.t. file naming conventions.
  3. Can run any number of products.
  4. For level II, by default, level II products are processed one VOS at a time; the granule input products are first degranulized. The algorithm programs take a single VOS product as input; thus, the script will send status messages for the singlevos processing. In contrast, specify '-g' to process level II using hourly granules as inputs to the algorithm programs; thus, the script will send a status message of processing for each granule file.
  5. Options are available to specify range of input files to be processed.
  6. Options, -b, and -c' should be omitted from the option file for level_1; note that the directory specified for option, -u, will be modified by this script.
  7. all_tape_level_1n2 uses the version numbers of TSDIS products from a file to construct products' filenames (File naming convention is specified by TSDIS). See option '-versions_file filename' for detail.
  8. all_tape_level_1n2 estimates the required disk space for running level 1 program or all of level 2 programs with the next input file. If there is no specified alternative directory, tt will stop and send the user a mail message indicating which file the program stopped at. This information can be used to resume the production.
Special Requirements:
  • The shell environment variables, GVS_DATA_PATH and TSDISTK, must be set.
  • 2A-53-w - Read gvs/src/2A-53/witek/README, section EXECUTION NOTES.
  • 2A-53-d - Read gvs/src/2A-53/danny/README, section EXECUTION NOTES.
  • 2A-53-v4 - Read gvs/src/2A-53/ver4/README, section EXECUTION NOTES.
  • 2A-53-dc - Read gvs/src/2A-53/dual-cappi/README, section EXECUTION NOTES.
  • 2A-54 - Read gvs/src/2A-54/README, section EXECUTION NOTES.
  • 2A-55 - Read gvs/src/2A-55/README, section EXECUTION NOTES.

all_tape_level_1:

all_tape_level_1 generates level I products for an entire 8mm tape. It can process data sets recognized by level_1 (This includes UF data set also). Darwin data set is assumed to contain tarred file(s) of compresssed file(s). Other data sets are assumed to contain compress/non-compessed file(s) and will be treated like Melbourne data set in production flow processing. The first file of tape contains site information; the second file, to the end of the tape, contains data. The first file is used for Melbourne data only. It is used as the 'first_filename_or_callid' parameter to level_1 for processing files 2 through the end of the tape. When processing Kwajalein or Darwin data, the first file is ignored. Typically the first file is a tape header file.

For data from tape, all_tape_level_1 stages the first file to disk and keeps it on disk until all remaining files on the tape are processed. In addition, while processing a particular input file the next input file from tape is staged, improving tape IO throughput. Once processing is completed, the input file is removed. After the last input tape file is processed, the first file that was staged is removed.

Note:

  1. This script can read data either from disk, CD, or tape. When reading data from disk, this program assumes data files are named $tape_id.#[.gz|.Z] or *.uf[.gz|.Z], where '#' is a sequential number starting from 0. Sequential number may either be a 3-, 4-, 5-digit, or non-leading-zero number. Darwin files are compressed tar archives and require one additional step to unpack the data. When reading data from CD, the input directory for non-Darwin raw data should contain files named $tape_id.#[.gz|.Z], where '#' is a sequential number starting from 0; the input directory for Darwin data should contain a tape header file named $tape_id.{0|000|0000|00000} and data files named *.vol[.gz|.Z]. With Berrimah data set either from tape or from disk, it contains no header file (The first file is data file) and an empty file between two data files. However, with Berrimah data set from CD, the header file should contain Berrimah. UF files of any site should be named *.uf[.gz|.Z].
  2. This script can process Melbourne, Kwajelein, Darwin, plus other data sets recognized by level_1. For production flow processing, this script treats the files tree of other data sets like the one for Melbourne data set. For product processing, the actual program 'level_1' will determine the data's site information.
  3. It checks to see if level I product file is a complete granule HDF file then it compresses that file. This checking is required since level_1 appends the result of the existing HDF file that belongs to the same granule.
  4. all_tape_level_1 estimates the required disk space for running the next file. If there is no specified alternative directory, tt will stop and send the user a mail message indicating which file the program stopped at. This information can be used to resume the production.
  5. all_tape_level_1 is able to process range of input files only.
  6. This script will add options '-f <first_filename>' appropriately depending on the site information in the first file when calling level_1; therefore, these options may be omitted from the option list. However, in some cases, the first_file does not contain the required information, the user can specify '-f <callID>'.
  7. Use absolute pathname when specifying '-b', '-c', '-g', and '-u' options; otherwise, $top_output_dir will be prepended to the pathnames.
  8. To specify the start or the last file, use the following convention:
    1. For UF data set for any site, use the file's name.
    2. For non-Darwin data set, use the file's name or sequential number.
    3. For Darwin data set, use the actual VOS filename for data on CD or use file's sequential number, $tape_id.$seq#, or $tape_id.$seq#[.gz|.Z]_tmp/$vos_fname for data not on CD.
    4. Note: Seq# does not include leading 0s if data is from tape.
level_3:

level_3 automates the processing of GV level 3 products for a whole tape. This program reads input from disk.

all_tape_2A-52:

all_tape_2A-52 automates the processing of GV's 2A-52i product for a whole tape. This program reads input from disk. There is a program that processes a whole month worth of data. It is called monthly_2A-52.

Special Requirements for All Programs:

The shell environment variables, GVS_DATA_PATH and TSDISTK, must be set.

Examples

Example 1:

If I have either Darwin, Kwajalein, or Melbourne data on an 8mm tape, how do I generate all of level I and II products?

Command: 

    all_tape_level_1n2 /dev/8mm1 N0001 level_1 level1.opts 
         2A-53-w 2A-53-w.opts 2A-53-c 2A-53-c.opts 
         2A-53-d 2A-53-d.opts 2A-53-v4 2A-53-v4.opts 2A-54 2A-54.opts 2A-55 2A-55.opts
    where:
    <product>.opts is a file containing product's options, listed in one line.
The above command will generate the following products: 1B-51 , 1C-51 , 2A-53-w, 2A-53-c, 2A-53-d, 2A-53-v4, 2A-54, and 2A-55 for tape named, 'N0001', residing on device '/dev/8mm1'.

all_tape_level_1n2 will prepend '-t L' to the option list for 2A-53-w if the data set is Darwin. It will append options, -c, -f (Melbourne only), and -b, to the option list for level_1. See the html documentation for each product for specific product options. Normally, though, you only need to specify '-v' as the only option to the program to get extensive diagnostic reporting during execution. For example, each *.opt file, above ,contains the string '-v' so that all defaults are used. Note, though, the *.opt file can be empty; it must exist, but it can be empty.

This script will create the following directories and will put the appropriate outputs there:

  • ./gvs_level_1_N0001/1C-51,
  • ./gvs_level_1_N0001/1B-51,
  • ./gvs_level_1_N0001/UF (if -u was specified in the option file for level_1),
  • ./gvs_2A-53-d_N0001,
  • ./gvs_2A-53-w_N0001,
  • ./gvs_2A-53-c_N0001,
  • ./gvs_2A-53-v4_N0001,
  • ./gvs_2A-54_N0001,
  • ./gvs_2A-55_N0001.
The processing follows these steps:
  1. Stage first file from tape to disk This is the tape header file. This file is used for Melbourne processing solely to determine the radar site id.
  2. Stage next file from tape to disk
  3. Run level_1, meanwhile staging the next file from tape to disk.
  4. Go to the previous step for the next file on disk
  5. Upon completion of level_1 processing for the whole tape, run level 2 on all the level_1 products. Processing of level 2 is at single VOS level since '-g' is not specified on the command line.
  6. The products are now on disk -- you can archive them to tape.

Example 2:

I have a Melbourne CD that holds three Melbourne tapes. The tapes on the CD are: kmlb001, kmlb002, and kmlb003. How do I generate all of level I products for the tape named, kmlb001?

Command: 

    all_tape_level_1n2 CD kmlb001 -raw_dir /cdrom/kmlb001 level_1 level1.opt
    where:
    level1.opt for level_1 must NOT include '-B -C'.
The above command will generate 1C-51 and 1B-51 for the Melbourne tape, kmlb001, that resides on the CD. The directory of the data is '/cdrom/kmlb001'. Note: '/cdrom' is a mounted point of a CD device.

all_tape_level_1n2 will create a directory, './gvs_level_1_kmlb001' and place the results there.

Example 3:

How do I generate all of level I products for a Darwin CD?

Command: 

    all_tape_level_1n2 CD darw001-1 -raw_dir /cdrom/ level_1 level1.opt
    where:
    level1.opt for level_1 must NOT include '-B -C'.
The above command will generate 1C-51 and 1B-51 for the entire CD having the first filename, darw001-1. The directory of the data is '/cdrom/'. Note: '/cdrom' is a mounted point of a CD device.

all_tape_level_1n2 will create a directory, './gvs_level_1_darw001-1' and place the results there.

Example 4:

I have either Darwin or Melbourne data on disk located in the current directory, which had been staged to disk from an 8mm tape named, W93-01. How do I generate 1C-51 products?

Command: 

    all_tape_level_1n2 disk W93-01 level_1 level1.opt
    where:
    level1.opt for level_1 must include '-B' .
The above command will generate only 1C-51, no 1B-51, for tape, W93-01, residing on disk located in the current working directory.

Note:

  • The input file names are W93-01.000 through W93-01.999. This is a bit extreme, typically the max is <tapeid>.250.
  • All input files can either be gzip-ed or ungzip-ed for Melbourne data. For Darwin data, the files must be a compressed tar archive.The first file, of both data types, must be uncompressed.
  • all_tape_level_1n2 will create a directory, 'gvs_level_1_W93-01' and place 1C-51 products there.

Example 5:

I have either Darwin or Melbourne data on disk. It had been staged to disk in directory, '/gvs/data/', from an 8mm tape named, W93-01. How do I generate 1C-51 and 1B-51 products?

Command: 

    all_tape_level_1n2 disk W93-01 -raw_dir /gvs/data/ 
          -o /gvs/productions/ level_1 level1.opt
    where:
    level1.opt for level_1 must NOT include '-B -C -b 1B_dir -c 1C_dir'.
The above command will generate 1B-51 and 1C-51 products for tape, W93-01, residing on disk located in '/gvs/data/'. 1B-51 and 1C-51 products will be placed in '/gvs/productions/gvs_level_1_W93-01/1B-51' and '/gvs/productions/gvs_level_1_W93-01/1C-51' , respectively.

Note:

  • The input file names are W93-01.000 through W93-01.999. This is a bit extreme, typically the max is <tapeid>.250.
  • All input files can either be gzip-ed or ungzip-ed for Melbourne data. For Darwin data, the files must be a compressed tar archive.The first file, of both data types, must be uncompressed.
  • all_tape_level_1n2 will create directory, 'gvs_level_1_W93-01' under '/gvs/productions/' and both '1C-51' and '1B-51' under it.

Example 6:

1C-51 products for tape, W93-01, had been generated; these products are still on disk. I want to generate 2A-54 products. How do I do that?

Command: 

    all_tape_level_1n2 disk W93-01 
         -qc_dir /gvs/productions/g    vs_level_1_W93-01/1C-51
         -o /gvs/productions/ 2A-54 2A-54.opt
    where:
    2A-54.opt contain only '-v '.
The above command will generate 2A-54 products under '/gvs/productions/' for tape, W93-01, using 1C-51 products located in '/gvs/productions/gvs_level_1_W93-01/1C-51'.

Note:

  • Since the input files to level II programs are expected to be on disk, the device is not necessary and can be anything. Something must be specified because this parameter is a place holder dummy argument. Specifying disk helps document the command.
  • all_tape_level_1n2 will use data in this directory, '/gvs/productions/gvs_level_1_W93-01/1C-51', for input to level II program.
  • all_tape_level_1n2 will also create directory '/gvs/productions/gvs_2A-54_W93-01' and place all outputs there.

Example 7:

Both 1C-51 and 2A-54 products for tape, W93-01, had been generated; they are still on disk. Now, how do I generate 2A-53-d products?
